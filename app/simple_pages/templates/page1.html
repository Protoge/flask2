OOP Glossary

Inheritance - In simplest terms, inheritance is concerned with the relationship between classes and method, which is like a parent and a child. A child can be born with some of the attributes of the parents. Inheritance ensures reusable of codes just the way multiple children can inherit the attributes of their parents.

Instantiation - In programming, instantiation is the creation of a real instance or particular realization of an abstraction or template such as a class of objects or a computer process.

Object - Object is an instance of a class; objects are the things you think about first in designing a program and they are also the units of code that are eventually derived from the process.

Class - A class is a template for creating objects in program. It is a logical entity.

Namespace - It is a declarative region that provides a scope to the identifiers (the names of types, functions, variables, etc) inside it.

Constructor - A special method of a class or structure in OOP that initializes a newly created object of that type.

Fixture - A sample data that allows you to populate your testing database with predefined data before your tests run. Fixtures are database independent and written YAML.

Type hint - Providing hints to function to only accept the given data type.

Type cast - A method of changing an entity from one data type to another.

Unit test - A method by which individual units of source code are tested to determine if they are fit for use.

Static - Static means no matter how many objects of the class are created, there is only one copy of the static member.

Instance method - An instance method is a method that belongs to instances of class, not to the class itself.

Instance property - A piece of data that is stored within an object.

Static method - A method that belongs to a class rather than an instance of class.

Encapsulation - In OOP, the notion of encapsulation refers to the bundling of data, along with the methods that operate on that data, into a single unit.

Polymorphism - Describes situations in which something occurs in several different forms.

Abstraction - The concept of OOP that "shows" only essential attributes and "hides" unnecessary information.

SOLID - It is an acronym for five principles that help software developers design maintainable and extendable classes. It stands for Single repsonsibility, Open-closed, Liskov substitution, Interface segregation and dependency inversion.

Design patterns - Concerned with the way of creating objects. These design patterns are used when a decision must be made at the time of instantiation of class.
