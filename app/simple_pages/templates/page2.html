Docker

Containers - A container is a standard unit of software that packages up code and all its dependencies so the application runs quickly and reliably from one computing environment to another.

Image - A container image is a lightweight, standalone, executable package of software that includes everything needed to run an application: code, runtime, system tools, system libraries, and settings.

docker run - run command is used to create a container from an image

docker build - build command is used to build an image from a specified docker file

docker ps - this command is used to list running containers

docker push - push command is used to push an image to the docker hub repository

How to start docker project: You can start the project by running the command docker compose up --build and you can access the container by docker exec -it container name
